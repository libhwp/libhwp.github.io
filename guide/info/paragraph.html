# 제어 문자

표, 그림 등 일반 문자로 표현할 수 없는 요소를 표현하기 위해서 문자 코드 중 일부분을 특수 용도로 사용하고 있습니다.

문단 내용 중에 문자 코드가 0-31인 문자들은 특수 용도로 사용됩니다. 특수 문자들은 표나 그림 등, 일반 문자로 표현할 수 없는 문서 장식 요소를 표현하기 위해서 제어문로 사용됩니다.

제어 문자는 다음과 같이 세 가지 형식이 존재합니다.

  • 문자 컨트롤 [char]: 하나의 문자로 취급되는 문자 컨트롤 (size = 1), libhwp에서는 enum class의 HWPControlCharType객체로 표현됩니다.
  • 인라인 컨트롤 [inline]: 별도의 오브젝트 포인터를 가리키지 않는 단순한 인라인 컨트롤 (size = 8), , libhwp에서는 enum class의 HWPControlInlineType객체로 표현됩니다.
  • 확장 컨트롤 [extended]: 별도의 오브젝트가 데이터를 표현하는 확장 컨트롤 (size = 8), libhwp에서는 enum class의 HWPControlExtendType객체로 표현됩니다.

WARNING

문서 파일에서 문단 내용을 읽다가 제어 문자를 발견하면, 문서를 읽는 쪽에서는 제어 문자 종류에 따라 읽어 들이거나 건너 뛰어 다음 데이터의 시작 위치까지 파일 포인터를 옮기기 위한 적절한 처리를 수행해야 합니다. 제어 문자 가운데는 또 다른 문단 리스트를 포함하는 경우도 있기 때문에, 제어 문자를 일반 문자처럼 처리하면 문서 파일을 정상적으로 읽을 수 없습니다.

# 문자 컨트롤

부가정보 없이 문자 하나로 표현되는 제어 문자입니다. (4번째. ch)

HWPCharControlChar 객체로 표현되며, Short타입의 code변수를 가지고 있습니다.

# 인라인 컨트롤

부가정보 12바이트(6 WCHAR) 이내에서 표현될 수 있는 제어 문자이다. info에 부가정보를 다 넣지 못하는 경우는 확장 컨트롤로 대체됩니다.(3-9까지의 8개의 ch)

HWPCharControlInline 객체로 표현되며, Short타입의 code변수와, 부가정보(info)를 담고 있는 12 bytes의 ByteArray타입의 addition배열을 가지고 있습니다.

# 확장 컨트롤

제어 문자는 포인터를 가지고 있고, 포인터가 가르키는 곳에 실제 오브젝트가 존재하는 제어 문자입니다. (3-9까지 8개의 ch)

HWPCharControlExtend 객체로 표현되며, Short타입의 code변수와, 포인터(pointer)를 담고 있는 12 bytes의 ByteArray 타입의 addition 배열을 가지고 있습니다. 이 addition을 활용하여 현재 문단안에 있는 컨트롤 객체들을 파악할 수 있으며, 반대로 컨트롤 객체를 생성하기 위해 addition값과 code값을 설정하여 추가할 수 있습니다. addition에 들어가는 컨트롤의 정보는 컨트롤 레코드 에서 확인할 수 있습니다.
