# 배포용 문서 데이터

Tag ID

HWPTAG_DISTRIBUTE_DOC_DATA

배포용 문서는 문서를 배포할 시 원본 문서의 일반적인 편집을 제한하기 위해서 설계된 기능입니다.

그 외에도 복사/붙이기, 인쇄를 제한하는 설정을 할 수 있습니다.

배포용문서와 일반 문서는 아래와 같은 차이점을 가집니다.

배포용문서 일반문서
본문 스트림 ViewText/Section BodyText/Section
암호화 여부 ⭕️ ❌
비고 복사, 인쇄제한을 위한 플래그

배포용 문서는 본문 스트림을 암호화하고 있으며, 이 외에도 아래와 같은 스트림을 암호화 합니다.

  • ViewText/Section0 ... N
  • Scripts/JScriptVersion
  • Scripts/DefaultScript
  • DocHistory/HistoryLastDoc
  • DocHistory/VersionLog0 ... N

# 데이터 표

문서 임의의 데이터을 나타냅니다.

libhwp에서 UnknownRecord객체로 표현됩니다.

자료형 길이 타입 변수 설명
BYTE array[256] 256 - - 배포용 문서 데이터
전체 길이 256

# 배포용 문서 레코드

배포용 문서 레코드를 나타냅니다.

자료형 길이 타입 변수 설명
WCHAR array[40] 80 - - - 배포용 문서를 생성시 사용된 암호의 SHA1해시코드
- 암복호화용 키 (처음부터 16바이트 까지)
WCHAR 2 - - 배포용 문서 옵션 flag
0x01 = 복사방지
0x02 = 인쇄 방지
전체 길이 82

# 배포용 문서 데이터 복호화

# Seed 찾기

복호화 과정에 필요한 Seed값은 배포용 문서 데이터의 처음 4바이트입니다.

자료형 길이 타입 변수 설명
UINT 4 - - Seed (배포용 문서 데이터의 첫 4바이트)
전체 길이 4

# 난수 배열 만들기

배포용 문서 데이터에서 해시코드를 추출하기 위해서는 우선 특정패턴을 지닌 난수배열을 생성해야 합니다. 이때 패턴을 생성하기 위해 MS Visual C의 랜덤함수(srand(), rand())를 사용합니다.

srand()의 초기 값은 앞서 찾은 Seed를 입력하며, rand()함수의 결과값에 따라 배열의 값이 채워집니다.

배열의 값이 채워지는 방식은 아래와 같습니다.

  1. srand() 초기화 (Seed 사용)
  2. rand()함수의 결과 값을 이용하여 배열을 채웁니다. 단, rand()함수가 호출되는 순번에 따라 그 사용 방식이 달라집니다.
    • 홀수번째: 배열이 채워지는 값
    • 짝수번째: 배열에 채워지는 횟수
  3. 홀수번째 rand() & 0xFF의 값을 A라고 하고, 짝수번째 (rand() & 0x0F + 1)의 결과를 B라 할 때 배열의 A값을 B번 횟수만큼 삽입합니다.
    • 예를 들어 A가 'a'이고 B가 3일 경우에 배열에 'a'를 3번 삽입합니다.
  4. 배열의 크기가 256이 될 때까지 1 - 3을 반복합니다.

# 해시코드 추출하기

난수 배열을 생성한 다음에는 배포용 문서 데이터를 이용하여 해시코드와 옵션 플래그를 생성할 수 있습니다.

해시코드와 옵션 플래그는 다음의 과정을 통해 생성됩니다.

  1. Seed로 부터 offset을 구합니다. offset은 다음과 값과 같습니다.
    • offset = (Seed & 0x0f) + sizeof(UINT)
  2. 난수배열과 배표용 문서 데이터를 XOR머지 합니다.
  3. XOR머지된 결과값에서 offset만큼 떨어진 위치의 80바이트가 해시코드이며, 그 다음 2바이트가 옵션 플래그입니다.

TIP

해시코드는 배포용 문서를 만들 때 사용된 암호를 SHA1 알고리즘으로 변환된 코드입니다.

해당 코드는 배포용 문서를 해제하거나 편집하는 용도로 사용됩니다.

# 해시코드와 AES-128 알고르짐을 이용하여 레코드 복호화

해시코드의 처음 16바이트는 본문 레코드를 복호화하는 용도로 사용됩니다.

본문 레코드는 AES-128 ECB를 통해 암호화 되었으므로 동일 알고리즘을 이용하여 복호화할 수 있습니다.
