# 데이터 타입

# 한글 문서에서 사용하는 자료형

자료형에서 한 바이트는 8 비트로 표현되며, 두 바이트 이상의 길이를 가지는 자료형은 아래 그림과 같이 최하위 바이트가 먼저 저장되고, 최상위 바이트가 가장 나중에 저장되는 리틀 엔디언(Little-endian) 방식입니다.

# 자료형 테이블

자료형 길이 부호 Kotiln Java 설명
BYTE 1 Short short 부호 없는 한 바이트(0~255)
WORD 2 Int int 16비트 컴파일러에서 unsigned int에 해당
DWORD 4 Long long 16비트 컴파일러에서 unsigned long에 해당
WCHAR 2 String String 한글의 기본 코드로 유니코드 기반 문자
HWPUNIT 4 Long long 1/7200인치로 표현된 한글 내부 단위
SHWPUNIT 4 ✔️ Int int 1/7200인치로 표현된 한글 내부 단위
UINT8 1 Short short unsigned __int8에 해당
UINT16 2 Int int unsigned __int16에 해당
UINT32(=UINT) 4 Long long unsigned __int32에 해당
INT8 1 ✔️ Byte byte signed __int8에 해당
INT16 2 ✔️ Short short signed __int16에 해당
INT32 4 ✔️ Int int signed __int32에 해당
HWPUNIT16 2 ✔️ Short short INT16과 같다
COLORREF 4 Long long RGB값(0x00bbggrr)을 십진수로 표시 (rr: red 1 byte, gg: green 1 byte, bb: blue 1 byte)
BYTE stream ByteArray byte[] 일련의 BYTE로 구성됨. 본문 내에서 다른 구조를 참조할 경우에 사용

# 추가 설명

WCHAR은 한글의 내부 코드로 표현된 문자 한 글자를 표현하는 자료형 입니다. 한글의 내부 코드는 한글, 영문, 한자를 비롯해 모든 문자가 2 bytes의 일정한 길이를 가집니다. 그렇기 때문에 Kotlin과 Java에서는 2 bytes의 바이트 배열을 생성 후, 스트림을 읽고 String으로 제공합니다.

HWPUNIT과 SHWPUNIT은 문자의 크기, 그림의 크기등 문서를 구성하는 요소들의 크기를 표현하는 자료형입니다. 문서 출력 장치 (모니터, 스마트폰 등)의 해상도는 가변적이기 때문에 dot의 수로 표현할 수 없습니다. 그렇기 때문에 한글은 1/7200의 기본 단위를 지정하여 사용합니다. 예를 들어 가로 3 인치 x 세로 2 인치의 그림의 크기를 HWPUNIT으로 표현하면 21600 x 14400으로 표현됩니다. 이 자료형을 이용하여 향후 pdf, docx, html등의 다른 문서포맷으로 변환 작업을 할 수 있습니다.
